Test For Success

Test Types
Development Testing
Testing done to understand the limits of the design space. Tests should be designed so that limits are reached about half the time to maximize information obtained. This way, design constraints are best defined. Sample sizes and test counts need to provide a good statistical basis for drawing conclusions.
Verification Testing
Testing done to verify that functional requiremenst are met. Test failures at this point should be rare and carefully evaluated if they occur.
Reliability Testing
Testing done to determine and/or verify expected product life. Testing must be done to failue to be sure that product strengths are well understood.
Statistical Terms
Statistical Significance
Statistical signifiance should always be considered when designing test plans and evaluating results. Consideration should be given to the following:
Sample Size (n)
Small sample size can contribute to design flaw escapes due to incorrect conclusions due to high Beta error. Beta error is inversly proportional to sample size.
Minimum (p-value) for rejecting the null hypothesis
This should be a business derive value where test cost is traded off for lower Alpha error.
